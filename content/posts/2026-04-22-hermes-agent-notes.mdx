上周看了 OpenClaw，这周量子位又出了 Hermes 的报道。两个在同一时间段冒出来，正好对比着看。

---

## Hermes 是什么

Python 写的开源 Agent 框架，核心设计理念是「让 Agent 越来越强」——也就是自我进化。

几个比较特别的设计：

**技能系统**：Agent 可以从 Markdown 文件里自动提取技能，不需要人工一条一条定义。你告诉它一个新操作怎么做，它自己把这个能力存起来，下次直接用。

**三层记忆**：内置记忆 + 外部存储 + 会话搜索，记忆体系比大多数框架完整。

**执行后端**：支持 6 种后端，包括无服务器选项。国内生态支持飞书、钉钉、企业微信。

**RL 工具链**：内置强化学习训练工具，可以对 Agent 的行为做反馈训练。

---

## 和 OpenClaw 的核心差异

OpenClaw 的逻辑是：把危险的口子堵死，再让 Agent 去执行任务。

Hermes 的逻辑是：让 Agent 自己判断风险，用辅助模型来评估操作是否安全，然后执行。

前者更保守，后者更激进。对于不需要生产级稳定性的个人用户来说，Hermes 的灵活度明显更高。

---

## 上手感受

Python 栈，文档还算清楚。最有意思的是技能自动提取这个功能——把一段教程丢给它，它真的能把步骤结构化成可复用的技能。

目前用它跑了几个文件处理的任务，速度和稳定性都还可以。RL 那一块还没深入，看起来是面向研究场景的。

---

现阶段 Agent 这个赛道还在混战期，OpenClaw 和 Hermes 代表两种不同的思路。如果是自己玩，Hermes 上手更快；如果是要接进公司系统，OpenClaw 的安全机制可能更合适。

先用着 Hermes，之后有深度体验再补。
